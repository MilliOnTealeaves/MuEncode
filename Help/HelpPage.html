Basics

Encoding or decoding a message

1. Enter your message:

  • Type your message into the textbox under ‘Input’
  • Click the ‘Paste’ button
  • Open a file—works best with text files

2. Select mode (encode/decode) based on your intended action

  • To encode a message, select ‘Encode’
  • To decode a message, select ‘Decode’

3. Choose a type of encoding

For more information, see ‘Encoding Types’ under ‘Detailed Information’

4. Select any options you want enabled

For more information, see ‘Options’ under ‘Detailed Information’

5. Click ‘Run’ to encode/encrypt your message

  • If you used AES encryption, make sure to save the key and initialisation vector if you elected to generate a new pair.
  • When done, you can copy the output to your clipboard via the ‘Copy’ button or by having ‘Copy result when done’ enabled in the Options panel, or save it to a file (not yet available)

6. Save your output!

  • Whether you just want to copy it or save it to a file, make sure you get your output stashed!

Detailed Information

Encoding Types

Morse Code

Usage information:

  • A space marks the end of a character
  • Forward slash (/) is used for space (Note: A space is needed on each side of the slash)
  • Accepts both uppercase and lowercase input, but does not preserve case (decoding will always result in lowercase text)

Example:
mu is a letter encodes to -- ..- / .. ... / .- / .-.. . - - . .-.

Currently supported characters:

  • English Letters A-Z
  • Numbers 0-9
  • Additional characters:
    . , ? ' ! / ( ) & : ; = + - _ " $ @

Random Shift

  • Translates message to morse code, with every character shifted ‘forward’ by a random number
  • E.g., with a shift of 3, a becomes d, p becomes s, etc.
  • Also applies to characters
  • Proprietary character order makes this harder to decode by hand

Repeated Random Shift

  • Iteratively uses Random Shift to encode a message
  • Decoding difficulty to decode is similar to Random Shift but takes longer

AES Encryption

  • Uses the AES standard and is interoperable with other AES software
  • Most secure option available in μEncode
  • Uses—or generates—a key and initialisation vector to encode any text
  • Symmetric encryption—key and IV needed on both sides of communication line

Usage information:

  • μEncode uses the key and IV currently entered in the textboxes that appear when AES is selected
  • If the key or IV field are left blank, or either of the two is invalid, a new pair is generated
  • Supports all UTF-8 characters

Options

‘Copy result when done’

Enabling ‘Copy result when done’ will copy the result of encoding/decoding to the clipboard after you click ‘Run’

‘Switch mode when done’

Enabling ‘Switch mode when done’ will make the mode switch from encoding to decoding, or vice versa, after input is processed.

This is useful in situations where one needs to encode messages to send and decode received messages in short succession
